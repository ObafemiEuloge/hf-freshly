GIT Learning

GIT est un VCS (Version Control System) ou en français un système de controle de version.

On distingue plusieurs types de systèmes:

  • Le système de gestion de version local
  • Le système de gestion de version centralisé
  • Le système de gestion de version distribué

GIT fonctionne avec des instantanés.

Un instantané est la sauvegarde de la version d'un fichier à un instant donné.

Un fichier peut être modifié ;

Un fichier peut être indexé ;

Un fichier peut être validé.

Il existe trois sections dans un projet GIT que sont :

  1. Espace de travail
  2. Zone d'index
  3. Repertoire GIT

Les commandes en général

cd -----------------------> pour changer de repertoire
dir ----------------------> pour lister tout ce qu'il y a dans un répertoire
cls/clear/CTRL+L ---------> pour supprimer

Les commandes en GIT

// Configuration globale pour tous les projets //

git config --global user.name "Votre nom d'utilisateur" -----------------> pour entrer le nom d'utilisateur
git config --global user.email "Votre adresse email" --------------------> pour configurer l'adresse email à associer

// Configuration pour un projet spécifique //

git config --local user.name "Votre nom d'utilisateur" ------------------> pour entrer le nom d'utilisateur
git config --local user.email "Votre adresse email" ---------------------> pour configurer l'adresse email à associer

==============================================================================================================================

git init -----------------------------------------------------------------> pour initialiser le repertoire
git add * / git add (nom du fichier) -------------------------------------> pour indexer le projet
git status ---------------------------------------------------------------> pour voir le statut de chaque projet
git commit -m "message" --------------------------------------------------> pour valider les fichiers indexés
git log" -----------------------------------------------------------------> pour voir toutes les sauvegardes
git commit --amend -m "Message" ------------------------------------------> permet de remplacer un fichier validé après indexation
git remote -v" -----------------------------------------------------------> permet de voir les serveurs auxquels le projet est connecté
git remote add nom_du_serveur adresse_du_serveur -------------------------> permet de voir les serveurs auxquels le projet est connecté
git remote remove nom_du_serveur -----------------------------------------> permet de supprimer un serveur relié à un projet
git push -u origin nom_de_la_branche -------------------------------------> permet d'envoyer un projet vers le serveur
